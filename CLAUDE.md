# CLAUDE.md

## Project Overview

BlackRock PDS (Private Data Service) Automation portfolio demonstrating workflow automation solutions using **VBA**, **Python**, and **SQL**. The project covers three independent but thematically linked components for investment data processing within the Aladdin Business ecosystem.

## Repository Structure

```
├── vba/                    # Excel add-in for SME loan portfolio validation
│   ├── modules/            # .bas VBA modules and .frm UserForm files
│   └── README.md
├── python/                 # Fund data ingestion & reporting pipeline
│   ├── sample_data/        # 3 CSV files from different GPs (test data with intentional errors)
│   ├── output/             # Generated reports, database, emails, logs
│   ├── config.py           # Centralized configuration (column mappings, FX rates, rules)
│   ├── pipeline.py         # Main orchestrator — runs all 5 stages
│   ├── ingestion.py        # Stage 1: Multi-format file reading
│   ├── validation.py       # Stage 2: Data quality checks
│   ├── standardization.py  # Stage 3: FX conversion, dedup, DB load
│   ├── reporting.py        # Stage 4: Excel report generation
│   ├── email_drafter.py    # Stage 5: LP notification drafting
│   ├── requirements.txt    # pandas>=2.0.0, openpyxl>=3.1.0
│   └── Readme.md
├── sql/                    # Database schema and recurring query templates
│   ├── schema.sql          # 3 tables: investments, validation_issues, ingestion_log
│   ├── queries/            # fund_performance.sql, portfolio_summary.sql, data_quality.sql
│   └── README.md
├── excel/                  # Pre/post-validation Excel workbooks (.xlsm)
└── README.md               # Main portfolio documentation
```

## Quick Start

### Python Pipeline

```bash
cd python
pip install -r requirements.txt
python pipeline.py
```

This runs all 5 stages sequentially and produces:
- `output/fund_data.db` — SQLite database
- `output/Quarterly_Report_Q1_2025.xlsx` — 4-tab Excel report
- `output/LP_Email_Q1_2025.txt` / `.eml` — LP notification email
- `output/pipeline.log` — Execution log

### SQL Queries

Queries in `sql/queries/` run against the SQLite database generated by the Python pipeline. Use any SQLite client or `sqlite3 python/output/fund_data.db`.

### VBA

Open `excel/SME_Loan_Eligibility_Checker_Input (1).xlsm` in Excel with macros enabled. The VBA source is in `vba/modules/`.

## Dependencies

- **Python**: pandas >= 2.0.0, openpyxl >= 3.1.0 (see `python/requirements.txt`)
- **Database**: SQLite (built into Python, no external DB needed)
- **Excel**: Macro-enabled workbooks (.xlsm)
- No external APIs or cloud services

## Code Conventions

### Python
- **Module organization**: One file per pipeline stage, orchestrated by `pipeline.py`
- **Configuration**: All mappings, FX rates, and rules centralized in `config.py` — no hardcoded values in stage modules
- **Logging**: Structured logging with both StreamHandler (stdout) and FileHandler (`pipeline.log`). Format: `"%(asctime)s | %(levelname)-8s | %(name)-20s | %(message)s"`
- **Error handling**: Try-except with safe parsing functions (`safe_parse`, `safe_float`)
- **Data processing**: Pandas DataFrames throughout; column renaming via config-driven mappings
- **Database access**: `sqlite3` with parameterized queries
- **Docstrings**: Triple-quoted module-level docstrings at top of each file

### VBA
- **Module organization**: Separated by concern (ValidationEngine, IntegrityChecks, Concentration, Dashboard, AuditExport, FXConversion, Ribbon)
- **Data structures**: Custom `Type` definitions (LoanRecord, EligibilityCriteria, ValidationResult, IntegrityIssue)
- **Performance**: Array-based batch read/write via `Range.Value`; `Application.ScreenUpdating = False` and `xlCalculationManual` during execution
- **Error handling**: `On Error GoTo ErrorHandler` pattern
- **Constants**: Row/column references as named constants, no magic numbers

### SQL
- **Style**: CTEs for complex queries, indented SELECT/FROM/WHERE/GROUP BY/ORDER BY
- **Null safety**: `NULLIF()` for division-by-zero protection
- **Formatting**: `ROUND(..., 0)` for monetary values, 4 decimal places for ratios
- **Comments**: `-- Description` at top of each file and inline for complex logic

## Architecture Notes

### Python Pipeline Stages
1. **Ingestion** — Detects GP source by column names, normalizes date/currency formats from 3 different GP data providers
2. **Validation** — 6 check types: missing fields, numeric validation, date validation, business rules, duplicates, format errors
3. **Standardization** — FX conversion to EUR, deduplication, status normalization, SQLite database load
4. **Reporting** — 4-tab Excel report (Summary, Fund Detail, Validation Issues, Ingestion Log) with formatting
5. **Email Drafting** — Templated LP notification in .txt and .eml formats

### VBA Validation Workflow
1. **Data Integrity** — 10 pre-validation checks on raw data
2. **Eligibility Criteria** — 9 configurable criteria per loan
3. **Concentration Analysis** — Portfolio-level limits (borrower, sector, country)
4. **Dashboard** — KPIs with conditional formatting
5. **Audit Trail** — Timestamped execution history

### Database Schema (SQLite)
- `investments` (14 cols) — Core fund data, PK: `investment_id`
- `validation_issues` (8 cols) — Data quality tracking with severity levels
- `ingestion_log` (5 cols) — Audit trail with auto-timestamping

## Testing

No formal test framework. Validation is done through:
- **Sample data** in `python/sample_data/` — 3 CSV files with intentional errors and varying formats
- **Excel workbooks** in `excel/` — 44 loan records covering edge cases
- **Pipeline output** — Check `pipeline.log` and generated reports for correctness

## Key Design Principles

- **Configuration-driven**: Validation rules, FX rates, column mappings, and thresholds are all configurable — not hardcoded
- **Audit trails**: Both VBA and Python components log all operations with timestamps
- **Multi-format tolerance**: Pipeline handles DD.MM.YYYY, MM/DD/YYYY, and YYYY-MM-DD date formats; multiple currencies
- **Separation of concerns**: Each module/stage has a single responsibility
- **No external dependencies beyond pandas/openpyxl**: Keeps deployment simple
